Commandes

Désactive un groupe entier ou des commandes une par une (slash + préfixe). help reste disponible sauf dans les salons marqués « sans commandes ». Les commandes modération / admin vérifient aussi les permissions Discord ; tu peux ajouter des rôles staff ou des restrictions ci‑dessous.

Commandes personnalisées

Réponses déclenchées par le préfixe (ex. $règles). Pas de slash Discord pour les perso.

Déclencheur : 1–32 caractères (lettres y compris accents, chiffres, _, -), nom différent d’une commande du bot. Il est enregistré en minuscules.

Mini-guide — texte dynamique dans la réponse

Placeholders (copie-colle dans la zone réponse) :

  • {{user}} — mention de l’auteur de la commande
  • {{username}} — pseudo Discord (sans mention)
  • {{user.tag}} — pseudo#0000
  • {{channel}} — mention du salon actuel
  • {{guild}} — nom du serveur
  • {{members}} — nombre de membres (approximatif)
  • {{reply}} ou {{reply.content}} — texte du message auquel tu réponds (commande en réponse)
  • {{reply.user}} — mention de l’auteur du message répondu
  • {{delete}} — supprime le message de commande après envoi de la réponse (le bot doit avoir « Gérer les messages » dans ce salon)

Exemple : Bienvenue {{user}} sur {{guild}} ! {{delete}}

Constructeur d’embeds

Rédige un message + embed comme sur Discord : aperçu en direct, mentions, champs, couleurs, puis publication ou mise à jour du message du bot dans le salon choisi.

Modération

Commandes slash et préfixe — permissions Discord requises

  • kick — Expulser (KickMembers)
  • ban — Bannir, option jours de messages effacés (BanMembers)
  • timeout / untimeout — Sourdine Discord (ModerateMembers)
  • warn — Avertissement + MP si possible (ModerateMembers)
  • slowmode — Mode lent du salon (ManageChannels)
  • clear — Purge de messages (ManageMessages)

Après mise à jour du code, redéploie les slash : npm run deploy (nécessite GUILD_ID dans .env).
